Getting Started

React 18 is the the latest version. Things have changed a bit in this release and here's how to get started:

npx create-react-app first_app

At this point, you should have a new React 18 project. You can see the result in the default browser if you start the server and head to the localhost.

npm run start
Component Based Design

React is all about writing small, reusable components. Each component can be defined as small part of a larger UI element. For example, buttons are used in forms, action buttons, cards and other places. Button, styled and ready, is a good candidate for making into a React component.

Some other UI components that can be good React component candidates are:

  • Navbar
  • Footer
  • ListContainer - fully ready major UI component
  • ListItem - individual reapeating entry in ListContainer
  • Content Card

Functional & Class Components

Class Components

We can use JavaScript ES6 syntax to model components using class syntax in JS. Here's a sample:

import { Component } from 'react';

class App extends Component {

  render() {
    return <div>Hello World!</div>;
  }
}

export default App;

Functional Components

Here's how you'd do the same thing using a functional component, which is the defacto standard today after hooks have made state management a breeze. 

const App = () => {
  return 
Hello World!
;
}

export default App;

As you can see, the functional variant is much more consise and easy to maintain.

React Hooks

React has a number of hooks built in and provides the user with the ability to write their owm custom hooks. All React hooks begin with `use` prefix. The two most important hooks in React are described below:

useState Hook

This hook replaces the need for class attributes in components by storing state.

const [value, setValue] = useState(initialValue);

useEffect Hook

The useEffect hook is used to execute enclosing code only when the specified dependencies change.

useEffect(() => { /* do something here */ }, [dependencies]);
